1. Connect to your repo

Needs Variables: Read & write permission on this repo. Generate one at github.com/settings/tokens (Fine-grained PAT recommended — restrict it to just this repo). The token stays in your browser's localStorage; nothing leaves your machine except the API calls to GitHub.

2. Runtime configuration

When OFF, every cron tick exits immediately. No posts, no Slack messages, no API calls. Re-enable any time.

The earliest cron tick after this hour generates the day's draft.

What this writes

Sets three GitHub Actions Repository Variables on your repo via the REST API: ENABLED, DAILY_DRAFT_HOUR, ENABLED_DAYS (or the STAGING_* equivalents). Your workflow YAML reads these on every run; the Python script gates the daily-draft phase on them.
